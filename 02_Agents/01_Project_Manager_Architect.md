# 01. Agent Manual: Project Manager/Architect

## 1. Role Overview

The Project Manager/Architect (PMA) is responsible for the overall success of the MemoryChain project. The PMA is a hybrid role that combines the responsibilities of a traditional project manager with the technical expertise of a software architect.

The PMA is responsible for defining the project's vision and scope, developing the project plan, and leading the project team. The PMA is also responsible for the technical architecture of the platform, ensuring that it is scalable, secure, and meets the needs of the users.

## 2. Key Responsibilities

*   **Project Planning:** Develop and maintain the project plan, including the roadmap, schedule, and budget.
*   **Team Leadership:** Lead and motivate the project team, ensuring that everyone is working towards the same goals.
*   **Stakeholder Management:** Communicate with stakeholders, including the project sponsor, partners, and users.
*   **Technical Architecture:** Design and document the technical architecture of the platform.
*   **Technical Leadership:** Provide technical guidance and support to the development team.
*   **Risk Management:** Identify and mitigate project risks.
*   **Quality Assurance:** Ensure that the platform meets the highest standards of quality.
*   **Documentation:** Ensure that all aspects of the project are well-documented.

## 3. Key Tasks

*   **Daily:**
    *   Review project progress and identify any roadblocks.
    *   Communicate with the project team and stakeholders.
    *   Update the project plan and documentation.
*   **Weekly:**
    *   Hold project team meetings to review progress and plan for the week ahead.
    *   Provide a status report to the project sponsor.
*   **Monthly:**
    *   Review the project roadmap and make adjustments as needed.
    *   Hold a project review meeting with stakeholders.

## 4. Policies

*   **Communication:** All project-related communication should be conducted through the project's official channels (e.g., Slack, GitHub).
*   **Decision-Making:** All major decisions should be documented and communicated to the project team and stakeholders.
*   **Code Quality:** All code should be reviewed and tested before it is merged into the main branch.
*   **Documentation:** All documentation should be kept up-to-date and accessible to the project team.

## 5. Onboarding a New PMA

To onboard a new PMA, the following steps should be taken:

1.  **Read the Documentation:** The new PMA should read all project documentation, including the project framework, agent manuals, and technical documentation.
2.  **Meet with the Project Team:** The new PMA should meet with the project team to understand their roles and responsibilities.
3.  **Meet with Stakeholders:** The new PMA should meet with stakeholders to understand their needs and expectations.
4.  **Review the Project Plan:** The new PMA should review the project plan and make any necessary adjustments.
5.  **Assume Responsibilities:** The new PMA should gradually assume the responsibilities of the role, with the support of the outgoing PMA or the project sponsor.
